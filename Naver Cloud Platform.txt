VPC(Virtual Private Cloud)
→ 클라우드상에서 논리적으로 격리된 고객 전용 네트워크 공간

10.0.0.0 ~ 10.255.255.255
172.16.0.0 ~ 172.18.255.255
192.168.0.0 ~ 192.168.255.255 중에 /28~/16으로 네트워크 대역을 생성

-------------------------------------------------------------------------------------
www.naver.com을 찾아가고자 할 때

1단계
- 1st DNS에 naver.com의 IP 주소를 물어본다
- 2nd DNS에 naver.com의 IP 주소를 물어본다
- 168.126.63.1 / 168.126.63.2  =>  KT

2단계
→ 168.126.63.1

3단계
→ 168.126.63.2

4단계
→ IANA → 최상위 도메인 관리 기관 → 전 세계의 모든 도메인 주소를 갖고 있다
→ naver.com의 주소는 125.209.248.6 이라는 것을 확인

5단계
→ 168.126.63.1 이 125.209.248.6한테 www의 IP 주소를 물어본다 그러면 naver.com 네임서버는 223.130.200.247이라고 알려준다 나에게 반환한다

6단계
→ 반환 받은 IP 주소로 접속한다

NS → 도메인을 위임 elec.naksoo
PTR → IP를 물어보면 www.naksoo.com을 알려준다
MX → 도메인이름     TTL     레코드타입     우선순위     메일서버도메인이름
      → 특정 도메인에 대한 이메일을 수신하는데 사용되는 메일 서버의 호스트이름을 지정한다

→ 커스터마이징 가능
→ 주기적으로 리포팅
-------------------------------------------------------------------------------------
WebShell Behavior Detector(VPC Only)
→ 외부로부터의 웹 쉘 공격을 실시간 탐지
→ Agent 기반
→ 행위 기반 실시간 웹쉘 탐지를 비롯하여 알려지지 않은 의심스러운 패턴까지 탐지
→ 웹쉘 의심 파일을 격리하거나 복구 가능

지원 환경
→ OS : Linux
→ WebServer : Apache, Tomcat, Nginx

Live Station
→ 실시간 방송을 위한 플랫폼
→ 트랜스코딩을 통해 여러 화질로 변환 후, 송출
→ 스트림 상태를 볼 수 있는 모니터링 기능 제공
→ Thumbnail Image 추출
→ 타임머신(Time Shift) 기능으로 놓치지 않는 라이브 방송 서비스 구현 가능
→ CDN 연동을 통해 안정적인 송출 가능 및 동시 송출
→ 녹화 저장(MP4, HLS)
→ restream 기능
-------------------------------------------------------------------------------------
VOD Station
→ Object Storage에 저장된 원본 영상을 모바일, PC 등 여러 디바이스에서 다양한 화질로 시청할 수 있도록 변환하는 서비스
-------------------------------------------------------------------------------------
Video Player Enhancement
→ video Player Enhancement는 웹/앱에서 비디오 또는 오디오와 같은 미디어를 재생할 수 있는 네이버 클라우드 플랫폼의 서비스
→ 플레이어 커스터마이징 : 기본으로 제공되는 플레이어에 원하는 옵션을 설정하여 플레이어 커스터마이징 가능
→ 옵션 설정 UI&UX 패널 : 콘솔에서 플레이어의 옵션 설정을 위한 UI&UX 패널을 제공하여 직접 플레이어 코드를 수정하지 않고 손쉽게 설정 가능
→ 스크립트 코드 : 콘솔에서 설정한 옵션에 따라 스크립트 코드를 제공하여 웹 페이지 내에 플레이어를 손쉽게 적용하고 개발 시간 단축 가능
→ HTML5 표준 제공
→ 컨텐츠 보안 
→ 모바인 SDK
→ CMAF LL-HLS 지원 : LowLatency HLS 플레이 가능
-------------------------------------------------------------------------------------
Media Connect Center
→ 네이버 클라우드 플랫폼 미디어 상품들을 하나의 통합된 환경으로 제공해주는 서비스
-------------------------------------------------------------------------------------
One Click Multi DRM
→ 컨텐츠에 DRM을 쉽게 적용하는 패키징 솔루션, 컨텐츠 재생을 위한 라이선스 발급 서비스, 안전한 재생 환경을 구축하는 클라이언트 연동 솔루션을 통합적으로 제공
-------------------------------------------------------------------------------------
NAVER WORKS
→ 네이버가 만든 업무용 협업 도구
→ 메신저를 중심으로 메일, 캘린더, 주소록, 설문, 드라이브 등 업무에 필요한 다양한 기능을 통합 제공하는 협업 도구

Hypervisor
Type 1 - native
→ 하드웨어 위에 하이퍼 바이저를 올리고 OS를 설치한다

Type 2 - hosted
→ 하드웨어 OS 위에 여러 하이퍼 바이저를 올리고 가상화된 OS를 설치한다
-------------------------------------------------------------------------------------
전가상화와 반가상화

전가상화
→ 하드웨어 전체를 가상화하는 방식
→ 하드웨어를 관리하는 하이퍼바이저(DOMO)가 VM들에게 하드웨어 리소스를 할당하는 방식

반가상화
→ 전가상화의 성능 저하를 해결하고자 모든 명령에 대해 DOMO를 통해 요청하는 방식이 아닌 하이퍼콜을 통해 직접 요청
→ OS 제약 발생
-------------------------------------------------------------------------------------
GPU
→ GPU 단위로 할당
→ Pass Through 방식으로 제공
  → 하이퍼바이저 환경에서 가상 머신이 호스트 시스템의 하드웨어 리소스를 직접적으로 액세스하는 기술을 가리킵니다.
-------------------------------------------------------------------------------------
Bare Metal Server
→ 하이퍼바이저 없이 하드웨어의 자원을 그대로 사용
  → 기본적인 관리는 IPMI를 통해 관리
→ 오라클과 같이 라이센스상 클라우드에 대한 제약기 있는 경우 사용
→ 서버 사양 및 지원 OS
  → RAID5, RAID1+0 지원
  → Classic 플랫폼 : CentOS, ORACLE Linus, Windows OS 제공
  → VPC 플랫폼 : CentOS, ORACLE Linus, RHEL, Windows OS 제공
