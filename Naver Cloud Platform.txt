상품 의존성
→ 클라우드도 물리적으로 어딘가에는 존재해야 한다

리전(Region)
→ 한국 리전, 일본 리전 
→ 나라를 뜻함

존(Zone)
→ 리전에 속한 한 구역
-------------------------------------------------------------------------------------
서버
Standard - 1:4
High CPU - 1:2
High Memory - 1:8
-------------------------------------------------------------------------------------
GPU 서버
T4 - 최대 2장의 GPU 제공
V100 - 최대 4장의 GPU 제공
A100 - Bare Metal Server로만 제공
-------------------------------------------------------------------------------------
국가별 지원 서버 타입(Region)
한국 싱가폴 일본 미국 독일 홍콩
-------------------------------------------------------------------------------------
VPC(Virtual Private Cloud)
→ 클라우드상에서 논리적으로 격리된 고객 전용 네트워크 공간
  - 현재 한국 리전 민간, 금융 클라우드에 적용
  - 계정당 최대 3개의 VPC 생성 가능
10.0.0.0 ~ 10.255.255.255
172.16.0.0 ~ 172.18.255.255
192.168.0.0 ~ 192.168.255.255 중에 /28~/16으로 네트워크 대역을 생성 
-------------------------------------------------------------------------------------
Peering
→ VPC간 연결을 위한 네트웍 구성
→ 내 VPC간 연결 뿐만 아니라 다른 계정과의 VPC 연결도 가능
→ 타계정 VPC 연결시 로그인 ID, VPC ID, VPC명 입력 필요
-------------------------------------------------------------------------------------
멀티테넌트
→ 단일 소프트웨어 인스턴스로 서로 다른 여러 사용자 그룹에 서비스를 제공할 수 있는 아키텍처를 말함
→ 서로 다른 고객이 서버 리소스를 나누어 사용하는 공유 호스팅을 멀티테넌시라고 부름
장점
→ 비용절감
→ 유연성
→ 효율성
-------------------------------------------------------------------------------------
VPC - Subnet

Public Subnet
→ 서버만 위치시킬 수 있으며 서버에 공인 IP를 부여할 수 있다.

Private Subnet
→ 서버 혹은 로드밸런서를 위치시킬 수 있다.

→ 서버넷은 존 안에 만들어진다
-------------------------------------------------------------------------------------
ACG & NACL (방화벽)
→ 방화벽이다.
→ 인터페이스(인바운드/아웃바운드)
→ 우선순위
→ Protocol / S-IP / D-IP / D-PORT / Action
→ 위 항목을 설정해야 한다

ACG 
→ 화이트 리스트 방식
→ 기본으로 모든 권한 불허
→ 서버 단위로 적용
→ Allow

NACL
→ 블랙 리스트 방식
→ 기본으로 모든 권한 허용
→ Subnet 단위로 적용
→ Action과 우선순위에 따라 규칙 적용
→ Allow / Deny
-------------------------------------------------------------------------------------
Init Script
→ 서버 생성 시 실행되는 스크립트를 지정
→ 서버 생성 시 최초 1회에 한하여 실행
→ 서버에 설치하여야 하는 패키지나 초기 설정 내용을 스크립트로 선언하여 서버 초기화를 최대한 빠르고 편리하게 구성

사용 용도
→ 같은 용도 서버를 여러 대 일괄로 생성하는 경우
→ 동일한 환경의 서버를 주기적으로 생성하는 경우
→ 용도별로 서버 초기 환경 관리가 필요한 경우
-------------------------------------------------------------------------------------
오토 스케일링
→ 인파리 확장의 높은 자유도를 이용하여 인프라 비용을 최적화하고 이벤트에 대응
→ 사용자가 정의한 주기(스케줄링), 사용자가 설정한 메트릭(모니터링), 사용자 요청(온디멘드)에 따라 서버를 자동 생성, 삭제 진행

→ Launch Configuration
→ Autoscaling Server group
→ Event Rule
-------------------------------------------------------------------------------------
컨테이너 서비스
→ Host OS 상에 논리적인 컨테이너를 만들고, 애플리케이션을 작동시키기 위해 필요한 라이브러리나 애플리케이션 등을 하나로 모아, 마치 별도의 서버인 것처럼 사용할 수 있게 만든 것
-------------------------------------------------------------------------------------
Load Balancer 
→ 네트워크 트래픽을 여러 대의 서버로 분산하여 부하를 균형있게 분담하는 장치나 소프트웨어를 말합니다
→ 요청을 처리할 대상에 대한 집합

설정
→ www.naver.com:443 Listen port
→ 타겟 그룹 port, Health check port

★ nslookup www.naver.com
-------------------------------------------------------------------------------------
www.naver.com을 찾아가고자 할 때

1단계
- 1st DNS에 naver.com의 IP 주소를 물어본다
- 2nd DNS에 naver.com의 IP 주소를 물어본다
- 168.126.63.1 / 168.126.63.2  =>  KT

2단계
→ 168.126.63.1

3단계
→ 168.126.63.2

4단계
→ IANA → 최상위 도메인 관리 기관 → 전 세계의 모든 도메인 주소를 갖고 있다
→ naver.com의 주소는 125.209.248.6 이라는 것을 확인

5단계
→ 168.126.63.1 이 125.209.248.6한테 www의 IP 주소를 물어본다 그러면 naver.com 네임서버는 223.130.200.247이라고 알려준다 나에게 반환한다

6단계
→ 반환 받은 IP 주소로 접속한다
-------------------------------------------------------------------------------------
DNS 레코드 타입

NS → 도메인을 위임 elec.naksoo
PTR → IP를 물어보면 www.naksoo.com을 알려준다
MX → 도메인이름     TTL     레코드타입     우선순위     메일서버도메인이름
      → 특정 도메인에 대한 이메일을 수신하는데 사용되는 메일 서버의 호스트이름을 지정한다
A → 도메인 이름과 해당 도메인의 IPv4 주소를 매핑하는 DNS 레코드
CNAME → 도메인 이름을 다른 도메인 이름으로 매핑하는 DNS 레코드

GTM(Global Traffic Manager)
→ 글로벌 규모의 DNS 기반 네트워크 트래픽 분산 처리 서비스
→ 전세계 어디든 naver.com을 접속 하려고 하면 krnaver.com / usnaver.com 가 아닌 naver.com으로 접속
→ Geolocation, Round Robin, Weighted, Failover
→ 최적의 루트, 한 단위씩 돌려서 서비스, 한 서버가 죽으면 다른 서버로 이전
-------------------------------------------------------------------------------------
CDN(Contents Delivery Network)
→ 컨텐츠를 Caching 하여 보다 빠르고 안정적으로 사용자에게 전송하는 서비스
→ CDN은 보통 T 단위로 구축한다 대역폭이 중요!
-------------------------------------------------------------------------------------
SSL VPN(Secure Sockets Layer Virtual Private Network)
→ 외부에서 네이버 클라우드 플랫폼 서버 접속 시 보안 통신이 가능하도록 보안 소켓 계층 기반의 가상 사설망
-------------------------------------------------------------------------------------
IPSEC VPN
→ 네트워크 망 vs 네터워크 망 연결
→ 고객의 사내망과 네이버 클라우드 플랫폼 간 사설 통신을 위한 IPSEC VPN
